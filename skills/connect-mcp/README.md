# connect-mcp

Подключает MCP-сервер к **Claude Code** «под ключ» — за пользователя и без терминала. Понимает три типа авторизации: без входа, OAuth (браузер открывается сам) и статический API-ключ (собирается приватно через `secret-input`, минуя чат). Работает на **macOS и Windows**.

Сделан для не-технарей: человек просто говорит «подключи вот этот MCP», а всю рутину (добавление, вход, ключ, проверку) делает агент.

## Что делает

1. Определяет тип по тому, что дал пользователь: URL `…/mcp` (HTTP), `…/sse` (SSE) или stdio-команда (`npx …`).
2. Спрашивает scope — во все проекты или только в текущий.
3. Добавляет сервер через `claude mcp add`, читает статус и ветвится:
   - **без авторизации** → сразу «Connected»;
   - **OAuth** → сам открывает браузер для входа, локальный колбэк ловит ответ;
   - **API-ключ** → собирает ключ через скилл `secret-input` (ключ не попадает в чат) и подставляет в конфиг, не печатая значение.
4. Проверяет подключение и предупреждает, что новые инструменты появятся в **новом** чате.

## Что нужно для работы

| Зависимость | Зачем | Когда обязательна |
|---|---|---|
| **Claude Code CLI** (`claude`) | скилл вызывает `claude mcp add / get / list / login` | всегда |
| **Python 3** | кросс-платформенный запуск OAuth-входа без терминала (`scripts/mcp_login.py`) | для OAuth-серверов |
| Скилл **[`secret-input`](https://github.com/MaxZarev/agent-mode-kit/tree/main/skills/secret-input)** | приватный сбор API-ключа, мимо чата | для серверов на статическом ключе |
| **Node.js** | нужен самому `secret-input` (локальная страница ввода) | транзитивно, вместе с `secret-input` |
| **winpty** (Windows) | выдаёт псевдотерминал для OAuth; идёт в комплекте с Git for Windows | только Windows + OAuth |

Собственных API-ключей у `connect-mcp` нет — он ничего не хранит, только настраивает подключение.

> ⚠️ **Зависимость от `secret-input`.** Ветка с API-ключом работает только если этот скилл установлен рядом. Ставь его так же, как `connect-mcp` (см. ниже).
> Ссылка: https://github.com/MaxZarev/agent-mode-kit/tree/main/skills/secret-input

## Установка

Скилл можно добавить в один из двух scope'ов (как и все скиллы этого набора):

- **User scope** — `~/.claude/skills/` — доступен во всех проектах.
- **Project scope** — `<проект>/.claude/skills/` — только в этом проекте.

```bash
# во все проекты (user scope)
cp -r skills/connect-mcp ~/.claude/skills/
cp -r skills/secret-input ~/.claude/skills/     # зависимость для ветки с ключом

# или только в текущий проект (project scope)
mkdir -p .claude/skills
cp -r skills/connect-mcp .claude/skills/
cp -r skills/secret-input .claude/skills/
```

После копирования перезапусти Claude Code, чтобы скилл загрузился.

## Как пользоваться

Просто попроси подключить сервер — скилл срабатывает на фразах вроде:

- «подключи mcp https://mcp.notion.com/mcp»
- «добавь коннектор sentry, я в терминале не шарю»
- «connect the linear mcp globally»
- «подключи наш внутренний mcp, там нужен api-ключ»

Дальше скилл сам спросит scope, откроет браузер для OAuth или страницу для ввода ключа и проверит подключение. Терминал трогать не нужно.

## Безопасность

- API-ключи собираются через `secret-input` и **не проходят через чат**; в итоге они лежат только в конфиге Claude Code (`~/.claude.json`), где нужны для подключения.
- Проверка серверов с ключом идёт через `claude mcp list`, а **не** `claude mcp get` — последний печатает заголовок с ключом открытым текстом.
- Перед установкой любого чужого скилла (включая этот) прочитай `SKILL.md` и `scripts/mcp_login.py`: это просто текст-инструкция и небольшой Python-скрипт, которые исполняются в твоей сессии.

## Как устроено

- `SKILL.md` — инструкция для агента (на английском): дерево решений, команды, работа с секретами и общение с не-техническим пользователем.
- `scripts/mcp_login.py` — запускает `claude mcp login` без интерактивного терминала: `pty` на macOS/Linux, `winpty` на Windows. Нужен потому, что `claude mcp login` сам по себе отказывается работать без TTY, которого у агента нет. Это не подменяет OAuth, а лишь даёт команде псевдотерминал.
