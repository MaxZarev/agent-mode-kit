# Платформы и запасной путь (fallback)

Скрипты этого скилла — **bash на Unix-инструментах** (`bash`, `git`, `python3`,
`perl`, `sed`, `find`, `sort`, `wc`, `tr`). Они нужны только для удобства: они
собирают пакет ревью и запускают внешние CLI с таймаутом. **Они не обязательны.**
Если на конкретной машине скрипты не запускаются, выполни те же шаги вручную
своими инструментами — не останавливай ревью.

## Как запускать скрипты на разных ОС

Определи окружение в начале запуска (например `uname -s`, наличие `bash` через
`command -v bash`), затем действуй так:

- **macOS / Linux.** Запускай скрипты напрямую по абсолютному пути или через
  `bash`:
  ```bash
  bash ~/.claude/skills/multi-review/scripts/build-code-review-pack.sh
  ```
- **Windows c WSL.** Это Linux — всё работает так же, как выше, внутри WSL.
- **Windows c Git Bash.** Запускай через `bash`. Учти две вещи:
  - Python может называться `python`, а не `python3` — обёртка
    `run-external-reviewer.sh` уже пробует оба варианта.
  - Убедись, что есть `perl` (для редактирования секретов) и `python`. Если их
    нет — см. запасной путь ниже.
- **Windows без bash (только cmd.exe / PowerShell).** Bash-скрипты тут не
  запускаются. Используй полностью ручной запасной путь ниже.

## Когда переходить на запасной путь

Переходи на ручной режим, если:

- `bash` недоступен (нативный Windows-шелл);
- билдер падает из-за отсутствия `perl` / `python` / Unix-утилит;
- `run-external-reviewer.sh` пишет статус `no_python`;
- любой скрипт завершается ошибкой, которую нельзя быстро устранить.

Принцип: **скрипты — это удобство, а не зависимость.** Ревью несколькими
моделями можно провести только своими инструментами.

## Ручной запасной путь

Скрипты делают четыре вещи. Сделай то же самое вручную.

### 1. Собери цель ревью

- **Режим кода:** определи охват сам обычными read-only командами git:
  ```bash
  git status --short
  git diff --name-only
  git diff main...HEAD --name-only   # если есть базовая ветка
  ```
  Запомни корень репозитория (`git rev-parse --show-toplevel`) как корень
  контекста.
- **Режим текста:** корень контекста — директория файла(ов), которые назвал
  пользователь. Прочитай эти файлы своими средствами.

### 2. Составь промпт ревьюера сам

Не нужен сгенерированный `prompt.md` — собери эквивалент в тексте, используя ту
же схему, что и скрипты (см. `references/prompt-code.md` и
`references/prompt-text.md`). Минимум:

- роль: «senior reviewer», работать **только на чтение**;
- корень контекста и список файлов / охват;
- применимые `AGENTS.md` / `CLAUDE.md` / `GEMINI.md`, если они есть;
- фокус ревью (баги/безопасность для кода; ясность/полнота/риски для текста);
- формат находок: `ID, Severity, Confidence, Area, Location, Impact, Evidence,
  Suggested fix`;
- правило «выводить только находки, без преамбулы».

Промпт к моделям держи на английском — так ревью кода стабильнее.

### 3. Не передавай секреты

Скрипты редактируют ключи и токены автоматически. В ручном режиме редактор может
быть недоступен, поэтому **не вставляй секреты в промпт сам**: давай ревьюерам
локальные пути, чтобы они открывали файлы у себя, а не вставляй чувствительные
куски в текст. Если без выдержки не обойтись — вручную замени ключи, токены,
пароли и строки подключения на `[REDACTED]`.

### 4. Запусти каждый CLI напрямую

Без обёртки. Один CLI на один отдельный вызов (не объединяй через `&&`):

```bash
# Codex
codex exec --skip-git-repo-check --cd "<корень-контекста>" "<твой промпт>"

# Claude
claude --add-dir "<корень-контекста>" -p "<твой промпт>"

# Gemini (через Antigravity CLI `agy`; промпт через stdin)
printf '%s' "<твой промпт>" | agy -p "Review the material provided on standard input and output findings only. Base the review solely on the supplied material; do not attempt to read other files."
```

- В Claude Code запускай каждый CLI отдельным вызовом инструмента `Bash` с
  `run_in_background: true` и `timeout: 900000` — таймаут и параллельность даёт
  сам харнесс, обёртка не нужна.
- Если фоновый запуск или таймаут недоступны в твоём окружении — запусти
  ревьюеров последовательно и просто дождись каждого. Это медленнее, но рабочий
  вариант.
- Считывай вывод каждого CLI сам и переходи к разбору находок.

### 5. Разбор и отчёт

Дальше — как обычно: разбери находки по `references/triage.md` и выдай отчёт.
Запасной путь меняет только способ сборки пакета и запуска CLI, но не правила
проверки находок.
