ℹ️ Описание — как пользоваться

🖱️Как менять. Страница сама ничего не меняет: строка — проект, колонка — сервер. Щёлкай по клеткам ✓/✕, нажми «Скопировать изменения» и вставь блок в чат агенту — он всё аккуратно применит. Наведи курсор на имя сервера в шапке — увидишь, как он подключён (stdio/http, команда или адрес).

🌐Строка «Все проекты». Верхняя строка переключает сервер сразу во всех проектах, где он есть; ниже — тонкая настройка по-проектно. Символ ± значит «включён не везде».

🔒Секреты. Значения API-ключей и токенов на страницу не попадают — видны только имена переменных.

🙈Лишние проекты. Скрывай крестиком в строке проекта — это тоже уедет в блок изменений, и в следующий раз они не покажутся.

🔌Коннекторы claude.ai. Появляются в списке после первого выключения через /mcp — реестра «включённых везде» коннекторов в локальном конфиге нет.

🖥️Режим Claude Desktop. Приложение управляет коннекторами claude.ai само и попроектные настройки не читает — в этом режиме их колонки свёрнуты и не переключаются. Работаешь в терминале или IDE — выбери «Claude Code (CLI)».

➕Сервер — в другой проект. У локальных и .mcp.json-серверов в чужих проектах стоит бледный «+»: клик — и агент подключит сервер туда (локальный — скопирует конфиг, .mcp.json — допишет в файл проекта и одобрит). Совсем новые серверы подключает скилл connect-mcp.

⛔Блокировки. ⛔ — сервер запрещён глобально (deniedMcpServers в ~/.claude/settings.json). Клик по ⛔ в строке «Все проекты» — и агент разблокирует его; после этого клетки проектов сразу можно донастроить. Приложение Claude Desktop эти блокировки для коннекторов не читает.

🧹Рудименты. Имена удалённых серверов, застрявшие в списках конфига. Разверни панель «Рудименты», нажми «очистить» — уборка уедет в тот же блок изменений.

✓ включён ✕ выключен ? ждёт одобрения (.mcp.json) ± включён не везде — не подключён + можно подключить (клик) ⛔ заблокирован — клик в строке «Все проекты» разблокирует 🖥 управляется в приложении Claude Desktop кольцо — твоё изменение
