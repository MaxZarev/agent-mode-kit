# mcp-dashboard

Показывает **все MCP-серверы во всех проектах** Claude Code на одной локальной HTML-странице — и даёт включать/выключать их по-проектно без ручной правки конфигов. Работает на **macOS и Windows** (и Linux).

Сделан для не-технарей: человек говорит «что у меня с MCP?» — агент открывает страницу-дашборд. Хочешь что-то поменять — щёлкаешь по ячейкам ✓/✕, жмёшь «Скопировать изменения», вставляешь блок обратно в чат — агент аккуратно применяет.

## Что показывает

Матрица «проекты × серверы»: **слева строки-проекты** (подтягиваются автоматически из конфига Claude Code), **сверху колонки-серверы**, сгруппированные по происхождению:

| Группа колонок | Откуда берётся |
|---|---|
| Глобальные (user scope) | подключены агентом глобально — доступны во всех проектах (stdio или http) |
| Встроенные | встроены в сам Claude Code (`claude-in-chrome`, `ide`) — доступны везде, выключаются по проектам |
| Коннекторы claude.ai | подключены на claude.ai / в Claude Desktop; появляются в списке после первого выключения через `/mcp` — реестра «включённых везде» коннекторов в локальном конфиге нет |
| Плагины | серверы, которые привезли с собой плагины Claude Code |
| Локальные (local scope) | подключены агентом к одному конкретному проекту |
| Проектные (`.mcp.json`) | лежат в репозитории проекта; видно, одобрены ли |

Верхняя строка **«Все проекты»** переключает сервер разом во всех проектах, где он есть (глобальное управление), строки ниже — тонкая настройка по-проектно. У локальных и `.mcp.json`-серверов в проектах, где они не подключены, стоит бледный «+» — клик, и агент подключит сервер туда (строка `add:` в блоке изменений): локальному скопирует конфиг из проекта-источника (секреты через чат не проходят — копирование идёт конфиг-в-конфиг), а `.mcp.json`-серверу допишет запись в файл `.mcp.json` целевого проекта и сразу одобрит — этот файл лежит в репозитории, так что изменение стоит закоммитить. Состояния: ✓ включён · ✕ выключен · ? ждёт одобрения · ± включён не везде (в строке «Все проекты») · — не подключён к проекту · ⛔ заблокирован глобально в `~/.claude/settings.json` (`deniedMcpServers`) — клик по ⛔ в строке «Все проекты» ставит разблокировку в блок изменений (`unblock:`), обратная операция `block:` доступна через агента («заблокируй X везде»); блокировки коннекторов приложение Claude Desktop не читает. Есть фильтр по имени сервера или проекта; как именно сервер подключён (команда/URL с замаскированными секретами) — в подсказке при наведении на имя сервера в шапке.

**Рудименты.** Имена серверов, которые давно удалены, но застряли в списках «включено/выключено» по углам конфига, больше не прикидываются настоящими серверами — они собираются в отдельную панель «🧹 Рудименты» с кнопкой «очистить» (уборка уезжает в тот же блок изменений строками `cleanup:`).

**Переключатель «Claude Code (CLI) / Claude Desktop»** (по умолчанию — Desktop, выбор запоминается). Приложение Claude Desktop берёт коннекторы claude.ai прямо с аккаунта и попроектные настройки не читает — переключать их из конфига бессмысленно. Поэтому в режиме Desktop колонки коннекторов свёрнуты в одну колонку «управляется в приложении Claude Desktop» (включать/выключать — значком инструментов в самом приложении или отвязкой коннектора на claude.ai). В режиме CLI коннекторы переключаются по проектам как обычно — в терминале и IDE это работает. Остальные группы серверов переключаются в обоих режимах.

**Проектов слишком много?** Скрой ненужные крестиком в строке проекта — выбор запомнится (в `~/.claude/mcp-dashboard.json`), и в следующие разы они не покажутся; вернуть можно из панели «Скрытые проекты». Появившиеся с прошлого раза проекты агент назовёт при запуске, а на странице они помечены бейджем «новый».

## Как меняет настройки

Страница **сама ничего не меняет** — она даже не имеет доступа к файлам. Кнопка «Скопировать изменения» кладёт в буфер текстовый блок:

```
=== MCP CHANGES ===
mode: cli
hide: /Users/x/old-experiment
unblock: claude.ai Gmail
project: /Users/x/proj
add: exa
enable: context7
disable: notion, claude.ai Gmail
cleanup: old-removed-server
=== END ===
```

(`hide:`/`show:` — скрытие/возврат проектов на дашборде; `mode:` — положение переключателя CLI/Desktop; эти строки меняют только state-файл скилла, не конфиг Claude Code. `cleanup:` — уборка рудиментов: имя вычищается из списков «включено/выключено» проекта; живой сервер так вычистить нельзя — скрипт откажется.)

Вставляешь его в чат — агент запускает скрипт, который:

1. делает **резервную копию** `~/.claude.json` (с меткой времени);
2. атомарно правит **только** MCP-ключи (`disabledMcpServers`, `enabledMcpjsonServers`/`disabledMcpjsonServers`) — ровно те же поля, что пишет встроенный тумблер `/mcp`;
3. ничего больше в конфиге не трогает.

Изменения действуют в **новых** сессиях Claude Code — уже открытый чат живёт со старым списком до перезапуска.

Если точно знаешь, что переключить («выключи context7 в этом проекте»), страница не нужна — агент составит блок сам и применит.

## Безопасность

- **Секреты не попадают на страницу и в чат**: значения API-ключей, заголовков и токен-подобные куски команд/URL маскируются (`•••`), видны только имена переменных.
- Страница — один локальный HTML-файл без сети и без внешних скриптов.
- Перед каждой записью — бэкап; запись атомарная (temp-файл + rename).

## Что нужно для работы

| Зависимость | Зачем |
|---|---|
| **Claude Code** | скилл читает его конфиг `~/.claude.json` (учитывает `CLAUDE_CONFIG_DIR`) |
| **Python 3** (3.9+) | скрипт парсинга/применения, только стандартная библиотека |

API-ключей у скилла нет — он ничего никуда не отправляет.

> Подключить **новый** MCP-сервер этот скилл не умеет (перенести уже настроенный локальный в другой проект — умеет, через `add:`) — для нового есть [`connect-mcp`](https://github.com/MaxZarev/agent-mode-kit/tree/main/skills/connect-mcp).

## Установка

Скилл можно добавить в один из двух scope'ов (как и все скиллы этого набора):

- **User scope** — `~/.claude/skills/` — доступен во всех проектах.
- **Project scope** — `<проект>/.claude/skills/` — только в этом проекте.

```bash
# во все проекты (user scope)
cp -r skills/mcp-dashboard ~/.claude/skills/

# или только в текущий проект (project scope)
mkdir -p .claude/skills
cp -r skills/mcp-dashboard .claude/skills/
```

## Использование

Просто скажи агенту:

- «что у меня с MCP?»
- «покажи дашборд MCP»
- «какие MCP включены в этом проекте?»
- «выключи context7 везде, кроме этого проекта»

## Файлы

| Файл | Что делает |
|---|---|
| `SKILL.md` | инструкция для агента |
| `scripts/mcp_dashboard.py` | `build` — собрать HTML-дашборд (+ детект новых проектов); `apply` — применить блок изменений (бэкап + атомарная запись) |
| `assets/dashboard.html` | шаблон страницы (данные подставляются при сборке) |
| `~/.claude/mcp-dashboard.json` | state скилла: скрытые проекты + известные проекты + режим CLI/Desktop (создаётся при первом запуске) |
