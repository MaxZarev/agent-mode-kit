# Agent Mode Kit

Набор материалов для работы с **Claude Code** в агентном режиме: глобальный `CLAUDE.md` Макса Зарева и коллекция из **34 скиллов**, которые он использует в работе и демонстрирует на курсе [**Агентный Режим**](https://zarev.academy).

Это не «учебная коллекция» — это реальный рабочий инструмент. Из него удалены только чувствительные данные (API-ключи, личные идентификаторы, упоминания закрытых клиентских проектов); личные пути и каналы заменены настройками в `.env`. В остальном — то, чем я сам пользуюсь ежедневно.

## Что внутри

| Что | Зачем |
|---|---|
| `CLAUDE.md` | Глобальный системный промт для Claude Code на русском — правила общения, работы с git, `.env`, скиллами, версиями. Подключается ко **всем** сессиям. |
| `skills/<name>/SKILL.md` | 34 готовых скилла — текстовые инструкции для Claude, которые он сам загружает при подходящем запросе. |

## Установка

### Глобальный CLAUDE.md

Скопировать в свой пользовательский конфиг (он применяется ко всем проектам):

```bash
mkdir -p ~/.claude
# Сохрани предыдущий CLAUDE.md если он есть!
[ -f ~/.claude/CLAUDE.md ] && cp ~/.claude/CLAUDE.md ~/.claude/CLAUDE.md.backup
cp CLAUDE.md ~/.claude/CLAUDE.md
```

⚠️ **Не заменяй вслепую.** Прочитай содержимое — там жёсткие правила (например, «всегда `git init` в новом проекте», «никогда не читать `.env`», «версии всегда проверять онлайн»). Возможно, что-то ты захочешь убрать или адаптировать под себя.

### Скиллы

Скиллы можно подключить в одном из двух scope'ов:

- **User scope** — `~/.claude/skills/` — доступны во всех проектах. Минус: каждый скилл добавляет описание в системный промт каждой сессии, увеличивая контекст. Используй только для скиллов, которые ты реально применяешь везде.
- **Project scope** — `<проект>/.claude/skills/` — доступны только в этом проекте. Лучший вариант для специфичных скиллов (например, «запостить в канал» — релевантен только там, где ты ведёшь контент).

```bash
# Глобально (user scope) — обычно НЕ рекомендую
cp -r skills/<skill-name> ~/.claude/skills/

# Локально в проекте (рекомендую)
mkdir -p .claude/skills
cp -r skills/<skill-name> .claude/skills/
```

После копирования перезапусти Claude Code или выполни `/skills` чтобы скилл загрузился.

**Не ставь все 34 сразу.** Выбери 3–5, которые закрывают твои реальные задачи, поставь их в project scope, поживи с ними неделю — потом добавляй следующие.

## Безопасность

Перед использованием любого чужого скилла (включая мои!) **проверь его содержимое**:

1. Открой `SKILL.md` и прочитай — что он делает, какие команды запускает
2. Загляни в сопутствующие скрипты (`scripts/*.py`, `scripts/*.sh`) — что они делают
3. Если скилл просит ввести API-ключ — он должен записывать его в локальный `.env`, а не передавать дальше

Скилл — это **текст**, который агент исполняет в твоей сессии. Если в скилле написано «удали всё в `~/` рекурсивно», агент попытается это сделать. Проверяй.

## Что про секреты

Все скиллы устроены так, что **API-ключи и токены живут только в локальном `.env` файле** рядом со скриптом. В коммитах этого репозитория ключей нет — `.env` блокирован в `.gitignore`. Если нужен ключ для какого-то сервиса — в `SKILL.md` или `.env.example` будет инструкция, где его взять. Для заполнения `.env` без вставки ключей в чат используй скилл `secret-input`.
